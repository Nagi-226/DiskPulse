# DiskPulse v0.2.0 — Performance & UX Optimization

> **Status**: Planning | **Target**: 2026-05-10 | **Base**: v0.1.0

## Problem Statement

v0.1.0 startup calls `scanDrive("C")` on mount, triggering a full recursive `WalkDir` scan of all top-level directories. On a typical 500GB Windows drive (500K+ files), this takes **10–30 seconds** before the user sees meaningful data.

### Root Cause Chain

```
App.tsx useEffect → scanDrive("C")
  → scanner::scan_drive_with_progress
    → scan_top_level_dirs (SEQUENTIAL)
      → for each dir: calculate_dir_size
        → WalkDir::new(path).into_iter().collect()  ← SYNC, FULL RECURSIVE
        → rayon par_iter for metadata               ← only metadata is parallel
```

1. **WalkDir is single-threaded** — walks one directory tree at a time
2. **Top-level dirs are sequential** — 8–12 dirs scanned one by one
3. **Full recursion** — every file in every subdirectory is visited
4. **No caching** — every startup is a cold scan
5. **UI blocks on scan** — the ring chart (which only needs GetDiskFreeSpaceExW) waits for directory scan

## v0.2.0 Roadmap

### Sprint 1: Instant Startup (2 days)

**Goal**: UI is interactive within 1 second, ring chart visible immediately.

#### 1a. Split scan_drive into two commands

| Command | Returns | Speed |
|---------|---------|-------|
| `scan_drive_meta(drive)` | `DriveMeta { total, used, free, cached_top_dirs }` | < 50ms |
| `scan_drive_dirs(drive)` | `Vec<DirInfo>` (existing, with progress events) | 5–30s |

`scan_drive_meta` calls:
1. `GetDiskFreeSpaceExW` → instant total/used/free
2. SQLite `get_snapshot_history(drive, 1)` → last cached top_dirs

#### 1b. Frontend: lazy loading pattern

```
Mount → show UI instantly → call scan_drive_meta → render ring chart
  → if cached data exists → show treemap with "cached" badge
  → auto-call scan_drive_dirs in background → progress bar
  → replace cached data with fresh scan
```

#### 1c. Remove blocking auto-scan

```diff
- scanDrive("C");  // blocks UI on mount
+ loadCachedDriveMeta("C");  // instant
+ setTimeout(() => scanDriveDirs("C"), 300);  // deferred background scan
```

### Sprint 2: Scanner Performance (2 days)

**Goal**: Full scan time reduced by 50–70%.

#### 2a. Parallel top-level directory scanning

```rust
// Before (sequential):
for entry in entries {
    let (size, file_count, dir_count) = calculate_dir_size(&path);
    dirs.push(DirInfo { ... });
}

// After (parallel via rayon):
let dirs: Vec<DirInfo> = entries
    .par_iter()
    .filter(|e| e.path().is_dir() && !is_protected_root_dir(...))
    .map(|entry| {
        let (size, fc, dc) = calculate_dir_size(&entry.path());
        DirInfo { ... }
    })
    .collect();
```

#### 2b. Progress: per-dir completion events

Instead of one progress per dir START, emit both start and completion:

```rust
#[derive(Serialize)]
struct ScanProgress {
    completed: usize,    // dirs finished
    total: usize,        // total dirs to scan
    current_path: Option<String>,
    // NEW:
    phase: ScanPhase,    // "walking" | "measuring"
    partial_results: Option<Vec<DirInfo>>,  // incremental results
}
```

Frontend updates treemap incrementally as each dir completes.

#### 2c. Use `jwalk` for parallel walking (optional)

`jwalk` parallelizes the directory walk itself (not just metadata):

```rust
use jwalk::WalkDir;
let results = WalkDir::new(path)
    .num_threads(8)
    .into_iter()
    .par_bridge()
    .filter_map(|e| e.ok())
    // ...
```

Tradeoff: adds dependency, but speeds up deep directory trees significantly.

### Sprint 3: Caching & Smart Refresh (1.5 days)

**Goal**: Subsequent launches show data instantly from cache.

#### 3a. SQLite cache on scan completion

After `scan_drive_dirs` completes:
- Save snapshot to SQLite (already implemented ✓)
- Also save to a `drive_cache` table keyed by drive letter (single row per drive, upserted)

#### 3b. Cache freshness indicators

| Cache age | Badge | Action |
|-----------|-------|--------|
| < 1 hour | "Live" (green) | No auto-rescan |
| 1–24 hours | "Cached" (yellow) | Background refresh |
| > 24 hours | "Stale" (red) | Prompt to rescan |

#### 3c. Background watcher integration

When the watcher is running and detects changes in a top-level directory:
- Mark that directory as "dirty"
- Re-scan only the dirty directory (not the whole drive)
- Update the affected DirInfo in the treemap

---

### Sprint 4: UI Polish (1 day)

#### 4a. Skeleton loading states

Replace "Scanning..." spinner with:
- Ring chart skeleton (animated pulse)
- Treemap skeleton (grid placeholder)
- Directory list skeleton (row placeholders)

#### 4b. Progressive scan display

- Ring chart appears instantly (meta data)
- Treemap tiles fill in as dirs complete
- Directory list sorts in real-time as sizes update

#### 4c. Cancel/abort scan

- User can click "Stop" to abort a long scan
- Uses cancellation token pattern (already in cleaner, reuse pattern)

## Technical Design

### New Tauri Commands

```rust
// Fast metadata + cache hit (replaces blocking scan on startup)
fn scan_drive_meta(drive: String) -> Result<DriveMeta, String>

// Existing full scan, now with incremental partial results
fn scan_drive_dirs(app: AppHandle, drive: String) -> Result<Vec<DirInfo>, String>

// Cancel running scan
fn cancel_scan() -> Result<(), String>
```

### New Types

```rust
#[derive(Serialize)]
struct DriveMeta {
    drive_letter: String,
    total_bytes: u64,
    used_bytes: u64,
    free_bytes: u64,
    cached_top_dirs: Option<Vec<DirInfo>>,  // from last snapshot
    cache_age_ms: Option<u64>,
}

#[derive(Serialize)]
enum ScanPhase {
    Walking,       // walking directory tree
    Measuring,     // computing sizes
    Complete,      // finished
}
```

### New Frontend Hook

```typescript
function useDriveScan(drive: string) {
  // Returns: { meta, dirs, progress, phase, isStale, rescan, cancel }
  // 1. Loads meta instantly
  // 2. Shows cached dirs if available
  // 3. Background scans for fresh data
  // 4. Updates incrementally
}
```

## Risk Assessment

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| jwalk adds compilation complexity | Medium | Fall back to rayon-only if jwalk integration fails |
| Parallel scanning causes I/O thrashing | Low | Cap rayon threads at 4 for dir scanning |
| SQLite cache stale after large deletes outside app | Medium | Show freshness badge; user can force rescan |
| Cancel mid-scan leaves partial state | Low | Clear partial results on cancel; no partial saves |
