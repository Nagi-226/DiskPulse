## v0.5.0 Release Plan

### Phase 1: Cross-Module Integration (v0.4.1 — v0.4.3)

> **Goal**: Wire together modules that were built independently. Fix the data flow gaps.

#### v0.4.1 — Recommendation + Disk Health Integration

**Task A: Wire aging data into recommendation engine**
- Call `aging::analyze_file_aging()` inside `get_recommendations()`
- Build path→age_days lookup map from aging report
- Populate `RecommendationInput.age_days` with real values
- Age scoring factor now uses actual file age instead of default 25%
- Add test: aged files score higher than fresh files

**Task B: Wire duplicate/zombie data into disk health**
- Call `duplicates::scan_duplicates_with_progress_and_cancel()` inside `get_disk_health()`
- Call `aging::analyze_file_aging()` inside `get_disk_health()`
- Sum total duplicate waste bytes from all groups
- Extract zombie total bytes from aging report
- Pass real values to `calculate_disk_health()`
- Add test: disk health score drops with high waste data

**Design decision**: `get_disk_health()` becomes a "full health check" — it will take 10-60s depending on drive size. This is acceptable for a health check command. Future optimization: background cache with configurable TTL.

| Metric | Before | After |
|--------|--------|-------|
| Health score factors | free_percent only | free_percent + growth + duplicates + zombies |
| Recommendation age scores | Always 25% (default) | Proportional to actual file age |
| Test count | 73 | 75+ |
