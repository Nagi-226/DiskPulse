### v0.7.3 — macOS 原生 CI + FSEvents (P0)

**目标**: macOS 平台全量构建/测试/打包 + 激活一直标记为 "pending native CI validation" 的 FSEvents 原生文件监控。

**FSEvents 激活**:

```
MacOsNativeWatcher:
  FSEventStreamCreate(paths, latency=1.0s, callback)
  FSEventStreamStart → 内核推送事件（<50ms 延迟, ~0% CPU）
  callback: 聚合事件 → 去抖 → emit(FsChangeBatch)
  stop: FSEventStreamStop + Release
```

| 对比 | 当前 Polling | 目标 FSEvents |
|------|-------------|---------------|
| 延迟 | 1500ms | <50ms |
| CPU 空闲 | ~2% | ~0% |
| 覆盖范围 | 仅监控目录 | 全目录树 |

**macOS 特有修复**: `osascript trash` → `NSFileManager.trashItem()` 主方案；过滤 `/System/Volumes/Data` 合成挂载点。

**制品**: `.dmg`

## Phase 2: 深度智能 (Deep Intelligence)

### v0.7.6 — 磁盘碎片分析 (P0)

**目标**: 全平台文件碎片度分析——不做整理，分析结果接入推荐引擎和健康评分。

**关键指标**: extent 数、碎片度 = `(extent_count - 1) / max(1, ceil(file_size / cluster_size))`

**三平台实现**:

| 平台 | API | 说明 |
|------|-----|------|
| Windows | `FSCTL_GET_RETRIEVAL_POINTERS` | DeviceIoControl → RETRIEVAL_POINTERS_BUFFER |
| Linux | `FS_IOC_FIEMAP` | ioctl → fiemap extent 数组 |
| macOS | `fcntl(F_LOG2PHYS)` | 逐块查询物理地址（大文件采样） |

**性能**: 大目录采样分析（最多 500 文件/目录），避免扫描百万文件时卡死。

**新 IPC**: `analyze_fragmentation(drive)`, `get_file_fragmentation(path)`, `cancel_fragmentation_scan()`

**新组件**: `FragmentationView.tsx` — 碎片化 Top-N 目录表格 + "关注此目录" 按钮

**接入**: 碎片度 > 50% → 推荐引擎加分（"高碎片目录，建议关注"）；碎片健康子维度接入 6D 健康雷达。

**涉及文件**: `src-tauri/src/platform/{windows,linux,macos}.rs`, `src-tauri/src/scanner/mod.rs`, `src-tauri/src/recommendations/mod.rs`, `src/components/FragmentationView.tsx`

### v0.7.7 — burn DL 异常检测引擎 (P0)

**架构**: burn 纯 Rust Autoencoder (6维→4维→6维) + Holt-Winters + Modified Z-Score → 三路信号融合器

**Autoencoder 设计**:

```
输入层 (6 维): daily_change_rate, file_growth_rate, waste_ratio,
               largest_dir_pct, day_of_week_embed, hour_bin_embed
Encoder:  6→16→8 (ReLU)
Bottleneck: 4 维 (潜在空间)
Decoder:  4→8→16→6 (ReLU)
输出: 重建误差 MSE(input, output)

异常判定: 重建误差 > 99th percentile 阈值 → AE 异常标记
推理时间: < 5ms (小型网络 + ndarray CPU 后端)
模型文件: diskpulse_ae.burn (~50KB)
```

**训练策略**: 合成数据生成器（1000 条正常 + 200 条异常）→ burn 训练 200 epochs → 预训练模型打包进二进制。用户可选本地 fine-tune（>60 快照后）。

**三路融合**:

| 检测器 | 权重 | 角色 |
|--------|------|------|
| Holt-Winters | 0.30 | 时序趋势 + 季节性 |
| Modified Z-Score | 0.30 | 统计离群点 |
| burn Autoencoder | 0.40 | 多维复杂模式 |
| 两路以上同时标记 | — | 置信度 HIGH |
| 单路标记 | — | 置信度 LOW |

**新增异常类型**: DriftAnomaly（多维漂移）、AntiSeasonal（反季节行为）

**新增 Crate**: `burn = "0.16"`, `burn-ndarray = "0.16"` (纯 Rust, 零 C 依赖, 二进制增量 ~500KB)

**涉及文件**: `src-tauri/src/anomaly/mod.rs`, `src-tauri/src/anomaly/ae.rs`, `src-tauri/src/anomaly/features.rs`, `src-tauri/src/anomaly/synthetic.rs`, `src-tauri/Cargo.toml`

### 🔒 burn 完整兜底预案

> 设计原则：任何 ML 组件故障不应导致 DiskPulse 核心功能降级。统计模型作为不可降级的基线。

#### 编译时兜底

```toml
# Cargo.toml
[features]
default = ["ml-engine"]           # Windows 默认开启
ml-engine = ["burn", "burn-ndarray"]
# cargo build --no-default-features → 纯统计模型，零 burn 依赖
```

```rust
// 所有 burn 代码受 #[cfg(feature = "ml-engine")] 保护
// feature off → 统计模型 100% 接管，功能无缺失（仅 DL 增强不可用）
```

#### 运行时兜底矩阵

| # | 风险 | 触发条件 | 兜底动作 | 恢复 |
|---|------|---------|---------|------|
| 1 | burn 编译失败 | `cargo build` 报错 | 该平台/CI 关闭 `ml-engine` feature | 等上游修复后重开 |
| 2 | 模型文件缺失 | `sha256(model) != expected` | `ae_disabled = true`, 日志 warning | 重新下载模型文件 |
| 3 | 模型文件损坏 | SHA-256 校验失败 | 同 #2 | 同 #2 |
| 4 | 推理超时 | `infer()` > 10ms | 跳过当前批次 AE，该批次仅统计模型 | 下一批次自动重试 |
| 5 | 模型精度退化 | 在线 AUC < 0.7 | AE 权重 0.4→0.1, 统计权重 0.6→0.9 | 本地 fine-tune 后恢复 |
| 6 | 合成数据不匹配 | 预训练模型在用户环境效果差 | 提供本地微调选项，默认关闭 | 积累 60+ 快照触发微调 |
| 7 | 用户拒绝 DL 功能 | Settings 关闭"AI 增强" | AE 权重 → 0.0, 纯统计模式 | 用户重新开启 |
| 8 | 二进制过大 | 产物 > 15MB | 模型文件不嵌入，首次从 GitHub Release 下载到用户数据目录 | 本地缓存 |

### v0.7.10 — 智能文件分类 (P1)

**从扩展名匹配到内容理解**:

三阶段分类流水线（类 duplicate detection 架构）:

```
Stage 1: 扩展名快速推断 → 匹配 80% 文件, < 1μs/文件
Stage 2: 魔数检测 (256B 文件头) → 匹配 15%, < 1ms/文件
Stage 3: burn 轻量分类器 (8 维特征) → 匹配 5%, < 5ms/文件
         ↓
  12 类输出: document_text/office/pdf, archive_compressed/installer,
             media_image/audio/video, dev_cache/build/dependency,
             system_db/config/temp, unknown
```

**burn 分类器**: Input(8)→Dense(32)→Dense(16)→Dense(12, softmax)。模型文件 `diskpulse_classifier.burn` (~80KB)。

**8 维特征**: file_size_log, has_extension, extension_entropy, byte_entropy_256, null_byte_ratio, printable_ratio, path_depth, parent_dir_type

**风险引擎增强**: 规则可添加 `file_category` 条件（如 "仅清理类别为 dev_cache 的文件"）

**前端**: LargeFileFinder 新增"类别"列 + 类别筛选器

**兜底**: Stage 1+2 覆盖 80% 文件。Stage 3 置信度 < 50% → 标记 Unknown（不参与自动清理）。burn 不可用时 → 仅 Stage 1+2。

## 风险评估

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| burn crate API 变更导致编译失败 | 中 | 中 | 锁定 `0.16` 版本；feature-gate 可在任何平台关闭 ml-engine |
| AE 合成训练数据不匹配真实场景 | 中 | 低 | 三路融合中 AE 权重可降级至 0.1；统计模型始终作为 fallback |
| 碎片分析需要管理员权限 | 高 | 低 | 复用 MFT 扫描的权限检测 + 降级逻辑；无权限→标记 unreadable |
| SignPath Foundation 审批不通过 | 低 | 中 | 备选 Sigstore (免费, GitHub OIDC) |
| FIEMAP 在老内核 (<4.11) 不可用 | 低 | 低 | 内核版本检测 + statx/stat 降级 |
| 10 个版本周期过长 | 中 | 中 | P0 优先 (7 个版本可先出 v0.8.0-rc), P1 渐进 |
| Homebrew Cask 入库需要知名度 | 中 | 低 | 入库前提供无签名 .dmg (右键打开) |

## 设计原则

1. **ML 不可降级核心功能** — 任何 DL 组件故障 → 统计模型自动接管 → 用户体验零影响
2. **安全红线不变** — 所有清理走 Recycle Bin，预清理必须用户确认
3. **纯 Rust 优先** — burn 而非 ONNX Runtime，零 C 依赖，跨平台编译一致
4. **渐进智能** — Stage 1-2 覆盖 80% → Stage 3 增强 15% → 兜底标记 5%
5. **三平台等权** — 新功能从设计阶段就考虑三平台实现，不事后补丁
6. **开源分发零成本** — SignPath ($0) + Homebrew ($0) + .deb/.AppImage (免费)
