### burn AE 异常检测 (v0.8.4, P0)

**架构**:
```
输入 (6 维): daily_change_rate, file_growth_rate, waste_ratio,
             largest_dir_pct, day_of_week_embed, hour_bin_embed
Encoder:  6→16→8 (ReLU)
Bottleneck: 4 维
Decoder:  4→8→16→6 (ReLU)
输出: 重建误差 MSE

异常判定: 重建误差 > 99th percentile → AE 异常标记
推理时间: < 5ms (ndarray CPU)
模型文件: diskpulse_ae.burn (~50KB)
```

**训练 pipeline**:
1. 合成数据生成器: 1000 条正常 + 200 条异常（基于真实快照统计分布）
2. burn 训练 200 epochs, batch_size=32, Adam optimizer
3. 预训练模型嵌入二进制
4. 可选: 用户本地 fine-tune（>60 快照）

**三路融合**:

| 检测器 | 权重 | 角色 |
|--------|------|------|
| Holt-Winters | 0.30 | 时序趋势 + 季节性 |
| Modified Z-Score | 0.30 | 统计离群点 |
| burn Autoencoder | 0.40 | 多维复杂模式 |
| 两路以上同时标记 | — | 置信度 HIGH |
| 单路标记 | — | 置信度 LOW |

**新增异常类型**: DriftAnomaly（多维漂移）、AntiSeasonal（反季节行为）

### burn 文件分类器 Stage 3 (v0.8.5, P0)

**架构**:
```
Input(8)→Dense(32)→Dense(16)→Dense(12, softmax)

8 维特征:
  file_size_log, has_extension, extension_entropy,
  byte_entropy_256, null_byte_ratio, printable_ratio,
  path_depth, parent_dir_type

12 类输出:
  document_text, office, pdf,
  archive_compressed, installer,
  media_image, audio, video,
  dev_cache, build, dependency, unknown
```

**训练数据**: 5000+ 样本（按文件类型分布合成）
**模型文件**: `diskpulse_classifier.burn` (~80KB)
**三阶段流水线**: Stage 1（扩展名, 80%）→ Stage 2（魔数, 15%）→ Stage 3（burn, 5%）

**风险引擎增强**: 规则支持 `file_category` 条件（如 "仅 dev_cache 类别自动清理"）

### 中继服务器 (v0.9.1, P0)

**设计原则**: 自托管优先 + 公共社区中继可选 = 零运维负担。

```
┌─────────────────────────────────────────────────┐
│              diskpulse-relay (独立 Rust binary)  │
│                                                  │
│  功能: WebSocket 中继 + 设备发现路由             │
│  部署: systemd unit / Docker / 手动运行          │
│  配置: 单个 YAML 文件 (端口、TLS 证书、速率限制) │
│  内存: < 20MB 空闲                               │
│  依赖: 零 (tokio-tungstenite)                    │
└─────────────────────────────────────────────────┘

工作流:
1. 用户可部署私有中继 (在家里的树莓派或 VPS)
2. 社区提供公共中继 wss://relay.diskpulse.dev（免费，无 SLA）
3. 两台设备连接同一中继 → 中继交换设备信息 → P2P 直连尝试 → 失败走中继转发
```

**安全模型**:
- 中继服务器**不解密**消息内容（端到端加密）
- 中继仅做路由转发，不存储数据
- 配对仍使用 6 位码（现有 Hub pairing 机制复用）
- TLS 1.3 传输层加密

### Cloud Sync Bridge (v0.9.2, P0)

**升级现有 Hub 协议**:

```
现有 (v0.7.0):

### Web Dashboard (v0.9.3, P0)

**架构：内嵌 HTTP 服务 + 同套 React Web 构建**

```
DiskPulse 桌面应用:
├── Tauri 前端 (React, 标准 Tauri 渲染)
│   └── 完整功能 (扫描、清理、设置)
│
├── 内嵌 HTTP 服务器 (axum / tiny_http)
│   ├── 绑定: localhost:PORT (仅本机)
│   ├── 静态文件: React Web 构建产物（同源码）
│   ├── WebSocket: /ws → 转发到 Hub
│   └── API: /api/* → 调用 Tauri 命令（只读白名单）
│
└── Hub (现有 WebSocket Server)
    └── 本地 + 远程设备数据聚合

**关键设计决策**:
- **一套 React 代码** → 编译两次：Tauri target (invoke IPC) / Web target (fetch + WS)
- Vite 双构建配置 (Tauri 模式 和 Web 模式)
- Web 模式下 `invoke()` polyfill → `fetch('/api/cmd/scan_drive_meta')`
- 类型共享（types.ts 一份）
- 安全模型：Web 模式不可执行清理（只读白名单）
